## 脚本 A · C 端：从一句话到委托律师（3 分钟）

**故事：** 跨境电商创业者想注册商标。

1. **登录** demo 账户，落地 `/dashboard`。
2. 右下角点击 **浮窗 AI**（FloatingAgent）→ 输入：
   > *"做跨境电商，刚给产品起了名字，想尽快注册商标。"*
3. 回车后跳转到 `/consult` 并自动触发匹配。
4. 画面展示：
   - **需求指纹卡**：意图=商标、紧急=紧急、地区=上海、关键词标签（trademark / cross-border / brand）
   - **Top 3 律师卡**：曹天明 (92 分) / 吴明昊 (84 分) / 苏棠 (76 分)
   - 每张卡含：匹配原因、评分、交付数、响应 SLA、推荐服务产品 + 价格
5. 点曹天明律师卡的 **「发起咨询」** → 跳 `/orders`（订单自动创建）。
6. 切到 `/match` → 查看匹配历史（含上次命中的画像快照）。
7. **收尾话术：** "从一句话 → 需求指纹 → 律师匹配 → 委托，全部由 AI 中枢调度完成；过程可追溯、可复盘。"

### 你要强调的技术亮点

- 需求画像不是 LLM 黑箱，而是规则+LLM 双通道抽取 → 可解释
- 匹配两阶段（规则召回 → LLM 重排）→ 降本 + 提质
- 每个律师卡都附带命中原因，直接回答"为什么是 TA"

## 脚本 B · B 端：律师的精准获客闭环（2.5 分钟）

**故事：** 曹律师登录自己的工作台，领取一条新线索。

1. 进入 `/provider`（导航项「律师工作台」）。
2. **工作台 Tab**：
   - 4 张 KPI 卡：30 天线索数、成交订单、成交金额、客户评分。
   - 收入品类条形图 + 漏斗：总线索 / 认领 / 成交。
   - 演示重点：**认领率** DonutRing + "3 小时内响应高温线索，可提升成交率 2~3 倍"。
3. 切到 **线索池 Tab**：
   - 3 条线索，分别 **热 / 温 / 凉**。
   - 每条含：客户名 + 行业 + 需求标签 + 匹配原因 + 画像快照。
   - 点击 **「立即认领」** 高温线索 → 状态立即变 "已认领"。
4. 切到 **我的产品 Tab**：律师上架的 3 个服务产品（全流程代办 / 驳回复审 / 异议答辩）。
5. 切到 **客户 CRM Tab**：已认领客户的 360° 画像（标签分组 + LTV + 订单）。
6. **收尾话术：** "从以前律师到处发名片，到现在 AI 精准派单；线索不是冷数据，是带需求指纹+命中原因的热线索。"

### 你要强调的技术亮点

- 匹配过程双向透明：C 端看到为什么推荐、B 端看到为什么匹配到我
- 线索质量温度（hot/warm/cool）由匹配分 + 意图 + 紧急度 + 画像完整度综合判定
- ROI 报表基于真实订单流水聚合，B 端可持续优化产品 & 定价

## 脚本 C · E 端：企业 IP 合规体检（2.5 分钟）

**故事：** 企业负责人要做一次 IP 体检。

1. 进入 `/enterprise`（导航项「合规中心」）。
2. **合规概览 Tab**：
   - 左侧 Donut：**62 分（橙色预警）**。
   - 右侧 5 维风险热力图：品牌保护 / 技术保护 / 软件版权 / 合同健康 / 政策敏感度。
   - 点 **「重新体检」** 按钮 → AI 重新跑一次体检，刷新分数与发现。
3. 下方 **3 条待处理发现**（每条带严重等级 + 描述 + 推荐服务产品 + 一键委托按钮）。
4. 切到 **体检报告 Tab**：带 **「下载报告」** 按钮（demo 用 markdown 输出）。
5. 切到 **政策雷达 Tab**：本行业政策 + 影响等级 + 施行日期 + 出处。
6. 切到 **订阅方案 Tab**：Free / Pro / Enterprise 三档，当前账户为 Free。
7. **收尾话术：** "AI 帮企业看到看不见的合规风险，每一条发现都能一键委托律师落地改善；不是卖 PDF，是卖持续的合规运营。"

### 你要强调的技术亮点

- 合规评分基于 IP 资产 + 合同 + 政策命中综合模型
- 发现 → 推荐服务产品 → 一键委托律师，形成"发现→成交"闭环
- 订阅分层让 SaaS 商业模型自洽，不是一次性工具

## 脚本 D · 诉讼智能预测（2.5–3 分钟 · 压轴）

**故事：** 创业者被指控商标侵权，不确定该应诉还是和解。

1. 进入 `/litigation`（导航项「诉讼智能」）。
2. 左栏点击 **Demo 场景「被诉商标侵权 · 电商类目」** → 表单自动填充（类型=商标侵权 / 角色=被告 / 证据分=3 / 对手=企业）。
3. 点击 **「AI 预测」**，中栏瞬间出现仪表盘：
   - **胜诉率 32%（高风险红色）** DonutRing
   - **赔偿区间 45 万~90 万** + 诉讼周期 150~300 天
   - **概率因子分解**：证据薄弱 -12%、对手规模大 -8%、管辖倾向中立、未做先期谈判 -5%
   - **策略卡**：① 优先和解 ② 启动证据公证保全 ③ 启动商标近似度专家意见
4. 切到右栏 **证据清单**，勾选 3 项必备证据（购销合同、公证书、专家意见）：
   - 页面 **不刷新**，左上角 DonutRing **从 32% 实时跳到 58%**，风险 Pill 变橙色
   - 中栏下方提示 **Δ +26%**（情景推演返回值）
5. 下方 **相似判例** 展示 3 条真实风格的 mock 判例（带法院 / 案号 / 年份 / 摘要）。
6. 右下浮窗 **FloatingAgent** 输入：*"这案子能打赢吗？能赔多少？"*
   - Agent 自动调用 `predict_litigation` 工具，再次给出结构化结论，并追加 follow-ups：
     "要我模拟补齐剩余证据后的胜率吗？" / "帮我找商标诉讼专家律师"。
7. **收尾话术：** "AI 把以前律师 1 周才能给出的诉讼评估，压到 3 秒内给出概率 + 金额 + 策略 + 类案；还能拖着滑杆看每一个动作对胜率的边际贡献 —— 这就是 AI 法务操作系统真正的代差。"

### 你要强调的技术亮点

- **确定性规则模型**：mock 预测可复现，保证演示不翻车；real 适配器预留 LLM + 裁判文书网接入点
- **端到端同构**：BFF mock 与后端 mock 共用同一套公式，前端 simulate 行为完全一致
- **Agent 深度集成**：不仅页面可用，用户在任何地方输入"能打赢吗"都能触发预测
- **场景化推送联动**：胜诉率 < 40% 自动推「和解建议」；≥ 70% 推「委托律师起诉」

## 通用问答备选（Q&A）

**Q1：你这个跟单点工具 SaaS 的差别是什么？**
> A1+ 2.0 不是 SaaS 工具，而是 AI 法务操作系统：AI Agent 做中枢调度，C/B/E 三侧共用同一画像与匹配引擎，服务数字化订单贯穿全程。单点 SaaS 只解决注册环节，我们解决"从需求到履约到续约"的全链路。

**Q2：AI 幻觉 / 给错法律意见怎么办？**
> 三重保险：① 所有 AI 输出带「仅供参考，以官方为准」；② 检测到高风险意图（诉讼/维权/合同争议）自动 `start_consultation` 转人工；③ 置信度低于阈值时走律师而非 AI 直接结论。

**Q3：真实数据 vs 模拟数据如何透明？**
> 每个响应都有 `mode: real | mock` 字段，前端不跨 mode 聚合。演示环境用 mock，生产 `.env` 切换。

**Q4：商业模式？**
> C 端免费用 AI + 抽佣（委托律师）；B 端线索订阅 + 品牌主页；E 端合规 SaaS 订阅（Free / Pro / Enterprise）；诉讼智能对接诉讼代理按件分成。三侧相互喂单，形成数据飞轮。

**Q5：诉讼胜诉率预测准不准？**
> 首版使用基于 IP 类案历史的**确定性规则模型**（类案基准 × 角色 × 管辖 × 证据 × 对手规模 × 谈判与专家加成），可解释、可复现，不做黑箱 LLM 推理；`RealLitigationPredictorAdapter` 已预留接入 LLM + 裁判文书网真实数据的接口，下一步升级为统计模型 + LLM 重排组合。所有输出均附「胜诉率与金额为 AI 模型预测，不构成法律意见」免责声明。
