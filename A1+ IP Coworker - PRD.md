## 一、产品概述

### 1.1 产品定义

**产品名称：** A1+ IP Coworker

**一句话定位：** 你的专属 IP 法务同事——懂法律、会主动、帮你跑腿。

**目标用户：** 中国小微创业者和 OPC（一人公司）

**核心使用场景：**
> 用户告诉 A1+ "我要注册商标"，A1+ 完成查重→分类建议→申请书生成→提交引导，用户只做最后的确认。

### 3.2 详细功能规格

#### F-01：IP 快速诊断

**触发方式：** 用户输入业务描述（文字/上传营业执照）

**处理流程：**
```
用户输入业务描述
    ↓
AI 分析业务类型
    ↓
判断应保护的 IP 类型（商标/专利/软著/版权）
    ↓
输出优先保护顺序
    ↓
生成《IP 保护建议报告》（PDF）
```

**输出格式：**
```
IP 保护建议报告
━━━━━━━━━━━━━━━━━━
诊断时间：2026-xx-xx

一、应保护资产（按优先级排序）
1. 商标「XX」- 第XX类 - 高优先级
2. 软件著作权「XX系统」- 中优先级
3. 技术专利「XX方法」- 低优先级（如有技术创新）

二、已识别风险
⚠️ 风险1：...
⚠️ 风险2：...

三、建议行动（按周排）
第1周：提交商标申请
第2周：进行软著登记
第3周：...

四、预估费用
商标注册：XXX元（官费+代理费估算）
...
━━━━━━━━━━━━━━━━━━
```

---

#### F-02：商标查重分析

**输入：** 商标名称 / 图形商标描述

**输出：** 冲突风险报告（分级：🟢可用 / 🟡近似 / 🔴冲突）

**技术路径：**
1. 商标局公开查询系统关键词检索
2. 搜索引擎全网近似商标检索
3. 电商平台同名店铺排查
4. 综合评分生成风险报告

**合规说明：** 仅使用公开可获取数据，不爬取无权限访问的内容。

---

#### F-03：商标申请书生成

**输入：** 商标名 + 申请人信息 + 类别选择

**输出：** 符合国家知识产权局规范的申请书文件（Word/PDF）

**文件包含：**
- 商标注册申请书（含填写模板）
- 类别选择建议及理由
- 商品/服务项目推荐清单
- 驳回风险提示
- 下证时间线预估

**质量要求：** 文件可直接提交至商标局电子申请系统

---

#### F-04：IP 资产台账

**用户操作：** 添加已有商标/专利/软著信息

**系统记录：**
| 字段 | 说明 |
|------|------|
| 名称 | IP 资产名称 |
| 类型 | 商标/专利/软著/版权 |
| 注册号 | 官方注册编号 |
| 申请日期 | 提交日期 |
| 到期日期 | 届满日期 |
| 状态 | 有效/待审/已过期/已续展 |
| 下次关键节点 | 续展/年费缴纳节点 |
| 相关文件 | 证书扫描件上传 |

**主动管理：**
- 到期前 90 / 60 / 30 天自动提醒
- 续展/年费到期前 7 天再次提醒
- 审查状态变更自动同步

---

#### F-05：侵权监控雷达

**监控范围：**
- 电商平台：淘宝/京东/拼多多/抖音小店 同名/近似商品
- 社交媒体：微博/小红书/抖音 疑似侵权内容
- App Store / 安卓应用市场 同名应用

**告警方式：**
- 微信公众号模板消息
- 微信客服消息
- 邮件（可选）

**告警内容：**
```
⚠️ 侵权线索发现
━━━━━━━━━━━━━━━━━
您的商标「XX」疑似被侵权：

平台：淘宝
店铺：XX旗舰店
商品：XX（与您的注册商标近似）
截图证据：已自动保存 [查看]
建议操作：[一键生成投诉模板]
━━━━━━━━━━━━━━━━━
```

---

#### F-06：竞争对手商标追踪

**配置方式：** 添加竞争对手公司名/品牌名

**监控内容：**
- 新注册商标（按类别）
- 商标申请状态变更
- 近似商标申请（与用户业务相关）

**推送频率：** 每周汇总报告 + 重大变更即时告警

---

#### F-07：合同 IP 条款审查

**使用方式：** 用户粘贴合同文本或上传 PDF

**分析维度：**
| 分析项 | 输出 |
|--------|------|
| IP 归属条款 | 归属方分析 + 风险评估 |
| 竞业限制条款 | 合理性评估 |
| 保密条款 | 完整性评估 |
| 侵权责任条款 | 责任分担评估 |
| 修改建议 | 具体修改文案建议 |

**输出形式：** 结构化报告 PDF + 可复制修改建议

## 四、非功能性需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | < 2秒（P95） |
| 商标查重响应时间 | < 30秒 |
| 申请书生成时间 | < 2分钟 |
| 告警推送到达率 | > 95% |
| 系统可用性 | 99.5%（月度） |

### 4.2 安全需求

| 需求 | 说明 |
|------|------|
| 数据加密 | 全站 HTTPS，用户信息加密存储 |
| 隐私合规 | 遵守 PIPL，不收集非必要个人信息 |
| 文件安全 | 用户上传的证书等文件单独加密存储 |
| API 安全 | 所有接口需认证，防爬防滥用 |

### 4.3 合规需求

| 需求 | 说明 |
|------|------|
| 代理边界 | 不替代用户进行任何形式的政府申报 |
| 数据来源 | 仅使用公开可获取数据，不爬取受限内容 |
| AI 透明度 | AI 生成内容均标注「仅供参考，以官方为准」 |
| 证据效力 | 侵权报告仅供用户参考，不构成法律效力证明 |

## 七、用户使用流程

### 7.1 主流程：商标注册

```
1. 用户注册/登录
       ↓
2. 进入「商标注册」模块
       ↓
3. 输入：商标名称 + 业务描述 + 申请人信息
       ↓
4. AI 查重分析（~30秒）
       ↓
   ┌→  🟢 可用 → 直接生成申请书
   ├→  🟡 近似 → 显示近似商标，给出建议 → 用户决策后生成
   └→  🔴 冲突 → 建议改名，给出备选方案
       ↓
5. 生成申请书 PDF/Word
       ↓
6. 用户下载 → 自行提交至商标局官网
       ↓
7. 系统自动添加至「IP 资产台账」
       ↓
8. 到期前自动提醒续展（系统自动管理）
```

### 7.2 主流程：IP 诊断

```
1. 用户输入业务描述（文字 or 上传营业执照）
       ↓
2. AI 分析应保护的 IP 类型和优先级
       ↓
3. 生成《IP 保护建议报告》
       ↓
4. 用户查看报告 → 选择要保护的类型
       ↓
5. 进入对应模块（商标/软著/专利）
       ↓
6. 继续注册流程（复用主流程）
```

## 九、FAQ（产品边界说明）

| 问题 | 答案 |
|------|------|
| A1+ 能帮我直接提交商标申请吗？ | 不能。我们做申请前的准备工作，提交需用户自行在商标局官网操作，这是合规要求。 |
| AI 生成的文件能被商标局接受吗？ | 可以。我们的模板基于官方规范生成，用户需核对后自行提交。 |
| 侵权监控的结果能直接当证据吗？ | 不能。侵权报告仅供参考，如需法律效力请通过公证处固证。 |
| AI 的建议构成法律意见吗？ | 不构成。我们的输出是信息分析，不构成律师法律意见。 |
